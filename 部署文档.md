## 一、部署概述

### 1.1 模块定位

本系统面向总部后台，承接城市合伙人（门店）提交的商品上架申请，由总部进行审核：
- **通过**：商品自动上架至「官方商品库」，并自动为对应城市建立「城市商品配置」。
- **驳回**：记录驳回原因，城市合伙人可修改后「重新提交」。
- 全程留痕：提交时间、审核时间、审核备注、关联商品 ID 均持久化。

## 四、审核账号

> 当前版本采用「角色 + 默认账号」模式用于部署验收，登录态由前端路由进入，暂未对接统一鉴权网关。
> 生产部署时建议在 Nginx/网关层对 `/api/store-products/:id/approve|reject` 等写接口增加鉴权。

### 4.1 角色定义

| 角色 | 权限范围 | 对应页面 |
| --- | --- | --- |
| 总部审核员 | 审核通过 / 驳回、删除记录、代城市合伙人提交、重新提交、统计查看 | 门店商品审核（全量） |
| 城市合伙人 | 新增门店商品申请、查看本人城市记录、驳回后重新提交 | 门店商品审核（按 cityId 过滤） |
| 系统管理员 | 分类、商品库、城市商品、推荐入口全量管理 | 全部模块 |

### 4.2 测试账号

| 账号 | 密码 | 角色 | 关联城市 | 用途 |
| --- | --- | --- | --- | --- |
| `admin` | `admin123` | 系统管理员 | - | 全量管理、部署验收主账号 |
| `reviewer` | `review123` | 总部审核员 | - | 审核通过/驳回主流程验收 |
| `hz-partner` | `hz123456` | 城市合伙人 | 杭州(city-5) | 非加盟城市提交验收 |
| `sh-partner` | `sh123456` | 城市合伙人 | 上海(city-2) | 加盟城市提交验收 |

> 说明：以上账号用于部署后的验收测试。`reviewer` 账号应可对 `spr-1001` 执行通过、对 `spr-1004` 执行驳回后重新提交；`hz-partner` / `sh-partner` 账号提交的新申请应出现在审核列表中，状态为「待审核」。

### 4.3 账号与数据隔离建议

- 城市合伙人仅能查看 `cityId` 等于自身的审核记录，前端通过 `filterCity` 过滤，后端建议在 [reviewService.findAll](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/023-总后台分类商品审核/server/services/reviewService.js#L77-L100) 增加按角色注入 `cityId` 过滤。
- 审核通过/驳回等写操作仅 `reviewer` / `admin` 可调用。

## 五、部署步骤

```bash
cd server
npm install
npm start
# 期望输出：[server] 分类管理后端已启动: http://localhost:3003
```

如需自定义端口：

```bash
PORT=3003 npm start
```

#### 开发模式

```bash
cd web
npm install
npm run dev
# 访问 http://localhost:5173
```

```bash
cd web
npm run build       # 产物输出至 web/dist

## 六、验收命令

> 以下命令假设后端已启动于 `localhost:3003`。请在部署机依次执行。

### 6.1 服务健康检查

```bash
curl -s http://localhost:3003/api/health | jq .
```

期望返回：

```json
{ "code": 0, "message": "ok", "data": { "status": "up", "time": "..." } }
```

### 6.2 接口验收

#### (1) 审核列表

```bash
curl -s "http://localhost:3003/api/store-products" | jq '.data | length'
# 期望：4
```

#### (2) 审核统计

```bash
curl -s "http://localhost:3003/api/store-products/stats" | jq '.data'
# 期望：{ "total": 4, "pending": 1, "approved": 2, "rejected": 1 }
```

#### (3) 按状态过滤（待审核）

```bash
curl -s "http://localhost:3003/api/store-products?reviewStatus=pending" | jq '.data | length'
# 期望：1
```

#### (4) 新建审核申请（代城市合伙人提交）

```bash
curl -s -X POST http://localhost:3003/api/store-products \
  -H "Content-Type: application/json" \
  -d '{
    "cityId": "city-5",
    "name": "验收用-招牌牛肉面",
    "categoryId": "cat-1-1",
    "icon": "🍜",
    "description": "部署验收测试商品",
    "price": 36,
    "originalPrice": 48,
    "tags": ["招牌"]
  }' | jq '.data.id'
# 期望：返回 spr-xxx 新 ID
```

#### (5) 审核通过（对 spr-1001）

```bash
curl -s -X POST http://localhost:3003/api/store-products/spr-1001/approve \
  -H "Content-Type: application/json" \
  -d '{ "reviewRemark": "验收通过" }' | jq '.data.reviewStatus'
# 期望："approved"
```

> 通过后可校验商品库已新增：`curl -s http://localhost:3003/api/products | jq '.data | length'`，数量应 +1。

#### (6) 审核驳回 + 重新提交（对 spr-1004）

```bash
# 驳回（spr-1004 当前已为 rejected，可先新建一条 pending 再驳回，或直接验证重新提交）
# 重新提交
curl -s -X POST http://localhost:3003/api/store-products/spr-1004/resubmit \
  -H "Content-Type: application/json" \
  -d '{
    "name": "翻新iPhone 15 Pro",
    "categoryId": "cat-4-1",
    "icon": "📱",
    "description": "已补充官方授权与质保凭证",
    "price": 6299,
    "originalPrice": 7999,
    "tags": ["特价"]
  }' | jq '.data.reviewStatus'
# 期望："pending"
```

#### (7) 删除审核记录

```bash
curl -s -X DELETE http://localhost:3003/api/store-products/<审核ID> | jq '.code'
# 期望：0
```
